    SyzygySys Terminology Reference & Security Dictionary

    This living document defines the core security, access, and identity concepts (AAA, IAM, PAM, RBAC, ABAC, OAuth2, OIDC, SAML, MFA, FGA) along with foundational SyzygySys, LAP/LAPI, and AI/LLM terms. Use this for consistent onboarding, documentation, and secure operations.


    Table of Contents

    • 🔐 Security, Access & Identity Terms
    • 🔧 LAP / LAPI & Integration Terminology
    • 🧠 AI / LLM Concepts
    • 🛰️ SyzygySys Terms & Project-Specific Language

    🔐 Security, Access & Identity Terms

    Acronym/Term Definition How it applies in SyzygySys/LAP
    AAA Authentication, Authorization, Audit (“Accounting”). Prove who you are, what you’re allowed, and log everything. All comms, agent actions, and user activity pass all three. Every login, every command, every HITL.
    IAM Identity and Access Management—users, roles, credentials, policies. Manages all human/agent identities in SyzygySys.
    PAM Privileged Access Management—controls and audits sensitive accounts/actions (admin, root, critical ops). All HITL and expansion actions are PAM-gated.
    PIM Privileged Identity Management—lifecycle/governance of high-trust identities. Tracks which users/agents have elevated permissions.
    RBAC Role-Based Access Control—access via assigned role (admin, contributor, observer, agent, etc.). All human and AI roles are policy-managed.
    ABAC Attribute-Based Access Control—access by properties (device, location, time). Combine with RBAC for context-aware permissions.
    SAML Security Assertion Markup Language—XML SSO standard. Useful for enterprise SSO integration, not default for FOSS.
    OAuth2 Delegated access tokens for APIs. Standard for SyzygySys/LAP app-to-app/agent auth.
    OIDC OpenID Connect—SSO/auth on top of OAuth2. Use for “login with …” style SSO when needed.
    MFA Multi-Factor Authentication—“something you know + something you have”. Required for HITL/admin and critical tasks.
    FGA Fine-Grained Authorization—per-resource or per-agent access. For detailed, per-project or per-agent permissioning.

    🔧 LAP / LAPI & Integration Terminology

    Term Definition
    LAP (Local Agent Proxy) Core system for local infra automation via secure, scriptable, agent-driven APIs. Modular, extensible, FOSS.
    LAPI LAP Integrations—pluggable modules that extend LAP for specific platforms (Jira, GitHub, Email, etc.).
    LAP::CORE Central FastAPI app in LAP; routes, authenticates, orchestrates all integrations.
    LAP::SECRETS Secrets management; securely loads/validates secrets from env/files.
    LAP::AUTH Auth layer for LAP API (token, basic, OAuth2, etc.).
    LAPI::GIT Integration for Git command execution and state validation via API.
    LAPI::JIRA Integration for Jira Cloud API (issues, epics, workflow sync).
    LAPI::SHELL Execute shell/bash commands from LAP API (for DevOps/automation).
    LAPI::OLLAMA Local LLM wrapper (Ollama, etc.) for agentic inferencing via LAP.
    LAPI::EMAIL Planned: send/receive/automate mail flows through LAP.

    🧠 AI / LLM Concepts

    Term Definition
    LLM Large Language Model—deep model for language understanding/generation.
    RAG Retrieval-Augmented Generation—combines LLMs with live data for informed responses.
    Agentic AI Semi-autonomous agents that act toward goals (e.g., tool use, planning, memory).
    Inference Running a trained model on new data to generate outputs.
    Token Smallest text unit for LLM processing; affects context/memory limits.
    OpenAI Agent Plugin- and goal-oriented agent framework (partially available as of 2025).

    🛰️ SyzygySys Terms & Project-Specific Language

    Term Definition
    SyzygySys Overarching system/brand for all projects (LAP, LAPI, ZOI, MCP, etc.).
    ZITH “Zerene In The House” — AI smart home presence (voice, context, automation).
    MCP (Model Context Protocol) Protocol for structured, agentic message passing. Enables A2A comms.
    A2A Agent-to-Agent comms—enabling independent agents to collaborate via structured messages.
    ZOI Zerene Orchestration Intelligence — a local LLM/AI orchestrator integrated with LAP/MCP.
    CHARIOT Mobile robotic base/platform for ZITH presence.
    Selflytics Ethical personal analytics/self-awareness via AI.
    Ztunez Personalized DJ agent (LLM + YouTube Music orchestration).
    LAP::MCP MCP integration for LAP—acts as node and gateway for agent/model comms.
    SDF — SyzygySys Diagnostic Framework Multi-level intelligent diagnostic & repair across services, networks, and agents.
    Sovereign Multi-LLM + RAG Node Private, on-prem multi-model inference with retrieval and policy controls.
    Reasoning Audit Trail (RAT) Audit/search/playback/rewind for agents and humans across the platform.
